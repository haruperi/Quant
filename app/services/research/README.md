# Research Edge Lab

The Research Edge Lab is a sandbox for performing statistical and machine learning studies to discover, calibrate, and validate trading edges. All findings, strategies, and calibration outputs are strictly read-only and flag as **advisory-only** to ensure zero interference with live risk profile engines and trading executions.

## Directory Structure

```text
app/services/research/
├── __init__.py           # Side-effect-free public exports
├── data.py               # Dataset preparation, cleaning strategies, quality reports
├── features.py           # Technical indicators, regime detectors, forward excursions
├── leakage.py            # Lookahead bias validation, chronological time-splits
├── metrics.py            # Standard metrics registry & calculators (Returns, Volatility, etc.)
├── helpers.py            # News window checkers, calendar parsers, resource limiters
├── reporting.py          # Markdown/JSON serialization, scorecard snapshots
└── studies/
    ├── __init__.py
    ├── eds.py            # Edge Discovery Studies (Mean Reversion, Trend Breakout)
    ├── null_models.py    # Bootstrapping, permutation tests, multiple comparisons correction
    ├── structure.py      # Swing points calibration, news-hypothesis evidence packaging
    └── unsupervised.py   # Optional PCA, clustering (K-Means) with sklearn shims
```

## Core Functionality

### 1. Data Cleaning & Enrichment
- **Timezone Normalization**: Automatically aligns timestamps to GMT/UTC.
- **Missing Bar Strategy**: Configurable missing-bar handling (forward fill, linear interpolation, drop, or none).
- **Spread Anomaly Capping**: Restricts spread spikes to a configured multiple of the median spread.
- **Quality Reports**: Produces a `DataQualityReportModel` highlighting cleaning actions and remaining issues.

### 2. Technical Indicators & Regime Detection
- **Statistical Features**: Z-scores, rolling percent rank, Bollinger Band %B, width, RSI, Momentum, Donchian channels, and ADR.
- **Hurst Exponent**: Pure Python/NumPy rescaled range estimator to gauge mean reversion vs. trend persistence.
- **Regime Detection**: Detects volatility regimes (ATR-based) and trend regimes (dual EMA crossover).

### 3. Leakage Control (Lookahead Bias Prevention)
- **Time-Split Enforcement**: Enforces chronological partition boundaries (train/validation/test) with a gap buffer to prevent overlapping lookahead leakage.
- **Feature Validator**: Rejects datasets if any features reference tomorrow's prices or are derived from lookahead horizons.

### 4. Statistical Validation & Multiple Comparisons
- **Block Bootstrapping**: Estimates confidence intervals for expectancy without assuming normal distributions.
- **Permutation Tests**: Computes empirical p-values by shuffling returns.
- **Multiple Comparisons Corrections**: Supports Benjamini-Hochberg (BH) and Holm-Bonferroni (HB) corrections to control the False Discovery Rate (FDR).

### 5. Unsupervised Modeling (Optional)
- Shims for **PCA** and **K-Means** clustering.
- Gracefully fails with a `ValidationError` when `scikit-learn` is missing, allowing all other studies to execute cleanly.

### 6. Scorecards & Reports
- Persists research snapshots to Markdown and JSON formats.
- Ensures atomic writes via temporary files and rename shims to prevent corruption.

## Safety & Governance

1. **Advisory Status**: Every recommendation generated by this module is marked as `advisory_only`. It cannot place trades or alter active live risk limits.
2. **Resource Boundaries**: Validates input size to prevent memory exhaustion, raising a `ValidationError` if the record count exceeds limits.
3. **No Side-Effects on Import**: Safe to import in any environment without performing file operations, network requests, or broker calls.
